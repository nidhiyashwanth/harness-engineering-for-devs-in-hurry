An entry file should route attention—not consume it.

The failure loop

1Mistake
2Add a rule
3Temporary fix
4More bloat

Context budget

10–20K tokens of instructions crowd out code, tool output, and reasoning.

Lost in the middle

Critical rules buried in long text are less likely to be retrieved and followed.

Priority conflicts

Hard constraints, guidelines, and old lessons look equally important.

Maintenance decay

Adding feels free; deleting feels risky. Signal-to-noise steadily declines.

Contradictions

Rules written at different times diverge, leaving the agent to choose.

Core concepts

A quick vocabulary map for remembering why routing beats accumulation.

Instruction bloat
Instructions consume 10–15% of the context window, leaving less room for code, tool output, and reasoning.
Signal-to-noise ratio
The share of loaded instructions that actually matter to the current task. Higher SNR means less irrelevant context.
Entry file
A short starting point that gives the overview, commands, global constraints, and routes to deeper documentation.
Reveal on demand
Load broad orientation first and detailed topic rules only when needed: frequent nearby, occasional tucked away, never-used omitted.
Can't tell what matters
When constraints and suggestions look identical, the agent cannot reliably distinguish red lines from preferences.

Keep truth near the code

Type definitions, interfaces, comments, and configuration explanations belong beside the implementation. The agent sees them while working, so do not duplicate them in AGENTS.md.

Refactor checklist

  • Audit instruction SNRMark relevance across five common task types.
  • Split into 3–5 topic docsKeep the entry file under 100–200 lines.
  • Convert historical notes to tests—or deleteDurable behavior beats narrative baggage.
  • Rerun the same task setCompare success and compliance rates.
  • Review regularlyRemove stale, redundant, and contradictory rules.
Takeaway

The entry file is a router, not an encyclopedia.
